Risks

We decided to use python to code our web scraping program which could potentially prove to be difficult as we are not sure if we have the required knowledge to make such a program. We could have issues with our code and not be able to correctly troubleshoot. Python may even turn out to not be the most effective tool to develop a web scraping program but at this point, we are not sure, so this is definitely a risk.

There are also ethical considerations to think about. It needs to be clear with what data we are actually scraping. Transparency with both users and the places that we are scraping data from should be fully aware of our use of the data. A risk that we also run is making sure the program is coded correctly to avoid any unexpected issues. An issue that is a particularly big risk is making sure we don't accidentally perform a denial-of-service attack (DoS). As web scraping programs commonly send the server hosting the site many requests to access its different data, this can use up the server resources and potentially lock out users from accessing the site which is not something we want to do.
